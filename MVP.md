# SEOhub MVP 方案

> 版本：v1.0（2026-04-23 首版，经 3 个 critic agent + 1 个 synthesizer agent 评审修订）
> 状态：**待用户确认第 2 章 7 项决策后进入 M1 开发**

---

## 1. 一句话描述

用 Claude AI 构建一个**中文内容站矩阵**，通过合规的搜索引擎优化吸引自然流量，经站内"推荐位"白帽导流至用户现有的**广告变现导航主站**。预算受控，运营工时每周 60–90 分钟，仅做守门人审批，不写内容。

## 3. 核心约束（硬约束，不可妥协）

1. **100% 白帽** — 无 cloaking / JS/Meta 自动跳转 / 门户页 / 链接农场 / scaled content abuse
2. **Cloudflare-only 基础设施** — 无 VPS、无自建服务器，降低运维
3. **海外托管 + 无 ICP 备案**
4. **用户不写内容、不润色文字** — 最多审批/拒绝 AI 产出
5. **所有 AI 调用经过 Budget 模块** — 无 bypass
6. **Day-1 完整数据模型** — 不在生产后迁移 schema
7. **所有出站指向主站的链接必须带 `rel="sponsored"` 且清晰标注"推荐"/"合作"**

## 4. 业务模型

### 4.1 流量漏斗

```
 搜索引擎(Bing + Google 主 / Baidu 被动)
          ↓  自然搜索流量
  子站矩阵 (M1: 1站; M6: 3站)
          ↓  推荐位 rel="sponsored"
     用户主站 (导航 + 20+ 原创长文)
          ↓  AdSense / 百度联盟
         广告变现
```

### 4.2 子站角色

- 每个子站 = 独立品牌 + 独立域名 + 单一垂直主题
- 目标：在低竞争长尾关键词建立主题权威，吸引自然流量
- 内容类型：**常青内容**（evergreen），禁止时效性新闻
- 变现路径：**推荐位导流主站**（非子站自身挂广告）

### 4.3 主站角色（项目扩展范围，见 D2）

- 顶部/首页：导航 + 广告商链接
- `/blog` 或 `/guides`：20+ 篇 1500 字以上原创长文（AI 生成 + 人工审批），主题为各子站垂类的"上游总纲"
- `/about`、`/author/*`、`/contact`、`/privacy`、`/terms`：AdSense 必需页面

## 5. 内容策略

### 5.1 单站单垂类原则

- 每站 ≥ 80% 文章落在该垂类核心语义图谱内
- 跨主题文章由 Sonnet 预判拦截，不写
- 杀掉原计划的"AI 自主跨主题选题"

### 5.2 5 个候选垂类（M1 必须锁 1 个）

| # | 垂类 | 目标人群 | Bing/Google 机会 | 百度风险 | AdSense 友好度 |
|---|---|---|---|---|---|
| V1 | **星座命理文化** | 中文年轻女性 | 中等，长尾巨量 | 低（时效零） | 高 |
| V2 | **复古游戏怀旧** | 中文男性 25-40 | 高，竞争少 | 低 | 中高 |
| V3 | 海外华人实用工具 | 海外华人 | 高，高商业意图 | N/A | 高 |
| V4 | 数码海外比价 | 跨境电商/海外华人 | 高 | 低 | 高 |
| V5 | 中文学习资源 | 华裔子女/学中文者 | 中，Google 主场 | N/A | 高 |

**首选 V1 或 V2**，M2-M3 再根据数据决定第 2 站。

### 5.3 内容产出量

- M1：10 篇 / 首站
- M2：累计 40 篇 / 首站
- M3：首站达 80 篇 + 第 2 站上线（10 篇）
- M4-M6：稳态 20 篇 / 月 / 站

## 6. 推荐位系统

### 6.1 位置与密度

每个子站页面可用的 10 个候选槽位（见 [ARCHITECTURE.md](ARCHITECTURE.md#推荐位引擎) 详细定义）。**同页面实际激活 ≤ 2 个 `rel="sponsored"` 指向主站**（文中 1 + 文末 1）。

**站级硬约束：**
- 指向主站的链接 ≤ 全站出站链接总数的 **18%**
- 仅 **70%** 文章挂推荐位，**30%** 页面纯净
- 主站锚文本 ≥ 10 个变体轮换，纯品牌词 ≤ 40%

### 6.2 UTM 追踪

每个推荐位带独立 UTM 参数（source / medium / campaign / content），低 CTR 位（< 0.5% 连续 14 天）自动下线。

### 6.3 Schema

推荐位区块用 `schema.org/RecommendationList` 标注，链接 `rel="sponsored nofollow"` 双重声明。

## 7. 运营模型

### 7.1 运维工时

**默认 60–90 分钟/周**，分配：
- 30 min：Veto 队列（Opus 预过滤 Top 10）批准 / 打回 / 删除
- 20 min：KPI 看板（收录、排名、CTR、UV 曲线）
- 15 min：GSC / Bing Webmaster 告警处理（死链、手动处罚）
- 10 min：预算告警确认

### 7.2 Veto 队列

AI 自评分 < 阈值的文章进入队列，Opus 4.7 再过滤到 Top 10（每周上限）。运营可：
- 批准（发布）
- 打回（重新生成，1 次）
- 删除
- 批量操作（同站点、同主题、同问题类型）

### 7.3 新站提案

AI 每两周提交 1-3 个新站垂类候选 + 5118/Bing 关键词数据。运营一键通过 / 拒绝，通过后自动：买域名 → 初始化仓库 → 部署首版 → 进入内容生产。

## 9. 里程碑 M1 → M6

| 里程碑 | 周数 | 范围 | 验收标准 |
|---|---|---|---|
| **M1** | W1-W4 | 1 站 + 10 文章 + IndexNow + GSC/Bing 接入 + CSV 成本账 + VCR CI | GSC 收录 ≥ 5 篇 / Claude 成本 < \$60 / 部署流水线绿 / 周运维 ≤ 90 min |
| **M2** | W5-W10 | 内容引擎（Queues+Workflows）+ Opus 预过滤 Veto + 主站 /blog 注入 20 篇 | 1 站累计 40 文章 / 主站 AdSense 初审通过 / 推荐位 CTR ≥ 2% |
| **M3** | W11-W16 | Next.js 仪表盘 + 预算告警 + Vectorize 去重 + 第 2 站上线 | 第 1 站 Bing+Google UV ≥ 800/月 / 无 Google 手动处罚 |
| **M4** | W17-W20 | 第 2 站补齐 + A/B 推荐位密度实验 + Schema 全覆盖 | 2 站合计 UV ≥ 2,500/月 / 实验给出最优密度 |
| **M5** | W21-W24 | 第 3 站上线（预算允许）+ 内链图谱自动化 | 3 站合计 UV ≥ 5,000/月 / AdSense 通过并产生收入 |
| **M6** | W25-W26 | 稳定化 + 成本复盘 + 下阶段决策 | 月收入 ≥ \$60 / 周运维稳定 ≤ 90 min |

每个里程碑结束必须产出：**KPI 数据快照** + **文档更新**（见 §11）+ **下阶段开工 checklist**。
